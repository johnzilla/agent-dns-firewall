1. Create `tests/match.test.ts` FIRST (red phase):
   - **exact match:** `malware.test` in index -> `isDomainInIndex("malware.test", index)` returns true
   - **subdomain match:** `malware.test` in index -> `isDomainInIndex("sub.malware.test", index)` returns true
   - **deep subdomain:** `malware.test` in index -> `isDomainInIndex("deep.sub.malware.test", index)` returns true
   - **label-boundary correctness:** `malware.test` in index -> `isDomainInIndex("notmalware.test", index)` returns FALSE (critical test)
   - **unrelated domain:** `isDomainInIndex("safe.example.com", index)` returns false
   - **TLD not in index:** `isDomainInIndex("test", index)` returns false (unless "test" was explicitly added)
   - **empty index:** `isDomainInIndex("anything.com", emptyIndex)` returns false
   - **buildDomainIndex:** creates a Set from string array, deduplicates

2. Create `src/match.ts` (green phase):
   - Export `buildDomainIndex(domains: string[]): Set<string>` -- simply `return new Set(domains)`
   - Export `isDomainInIndex(domain: string, index: Set<string>): boolean`:
     - Check exact match: `if (index.has(domain)) return true`
     - Suffix walk at label boundaries: find each `.` position, extract parent domain after it, check `index.has(parent)`
     - Return false if no match found
   - This is the algorithm from the research doc -- walk up the domain hierarchy at dot boundaries only.
  </action>

<task type="auto" tdd="true">
  <name>Task 2: Decision logic with precedence, input sanitization, and public API</name>
  <files>src/decide.ts, src/index.ts, tests/decide.test.ts</files>
  <action>
1. Create `tests/decide.test.ts` FIRST (red phase). Tests use a helper that builds a full decision context (allowSet, denySet, blocklist entries) and calls the exported `isDomainBlocked`:

   **BlockDecision shape (MATCH-03):**
   - Blocked by blocklist returns `{ blocked: true, reason: "blocklist", listId: "test-list" }`
   - Blocked by deny returns `{ blocked: true, reason: "custom-deny" }` with no listId
   - Not blocked returns `{ blocked: false }` with no reason or listId

   **Allow override (OVER-01):**
   - Domain in allow list AND in blocklist -> NOT blocked
   - Domain in allow list AND in deny list -> NOT blocked
   - Allow uses exact match only: `google.com` in allow does NOT allow `ads.google.com`

   **Deny override (OVER-02):**
   - Domain in deny list but NOT in allow list -> blocked with reason "custom-deny"
   - Deny uses exact match only: `evil.com` in deny does NOT block `sub.evil.com`

   **Precedence (OVER-03):**
   - Domain in allow AND deny AND blocklist -> NOT blocked (allow wins)
   - Domain in deny AND blocklist -> blocked with reason "custom-deny" (deny checked before blocklist)
   - Domain in blocklist only -> blocked with reason "blocklist"
   - Domain in nothing -> not blocked

   **Resilience (RESL-02):**
   - `isDomainBlocked(null as any)` -> `{ blocked: false }`
   - `isDomainBlocked(undefined as any)` -> `{ blocked: false }`
   - `isDomainBlocked("")` -> `{ blocked: false }`
   - `isDomainBlocked(123 as any)` -> `{ blocked: false }`
   - `isDomainBlocked({} as any)` -> `{ blocked: false }`
   - Never throws for any input

   **Input sanitization:**
   - `isDomainBlocked("https://evil.com/path")` extracts hostname `evil.com` and checks it
   - `isDomainBlocked("evil.com:8080")` strips port and checks `evil.com`
   - `isDomainBlocked("EVIL.COM")` normalizes to lowercase before checking
   - IP address input (`"192.168.1.1"`) is checked as a normal string against the index

   **Blocklist source identification:**
   - When blocked by first source, listId matches that source's id
   - When blocked by second source (not first), listId matches second source's id

2. Create `src/decide.ts` (green phase):
   - Import `normalizeDomain` from `./normalize.js`
   - Import type `BlockDecision` from `./types.js`
   - Import `isDomainInIndex` from `./match.js`
   - Export `sanitizeInput(input: unknown): string | null`:
     - Guard: if input is null, undefined, or not a string, return null
     - Trim; if empty, return null
     - If contains `://`, try `new URL(input).hostname` (catch and continue if fails)
     - Strip port: find last `:` after last `]` (IPv6 safety), slice before it
     - Return `normalizeDomain(result)` or null if empty
   - Export `isDomainBlocked(input: unknown, allowSet: Set<string>, denySet: Set<string>, blocklistEntries: Array<{ index: Set<string>; sourceId: string }>): BlockDecision`:
     - Wrap entire body in try/catch returning `{ blocked: false }` on any error
     - Call sanitizeInput; if null, return `{ blocked: false }`
     - Check allowSet.has(domain) -- exact match. If allowed, return `{ blocked: false }`
     - Check denySet.has(domain) -- exact match. If denied, return `{ blocked: true, reason: 'custom-deny' }`
     - Loop blocklistEntries, call isDomainInIndex(domain, entry.index). If match, return `{ blocked: true, reason: 'blocklist', listId: entry.sourceId }`
     - Return `{ blocked: false }`

   - Re-export decision: `export { sanitizeInput, isDomainBlocked } from './decide.js'`
