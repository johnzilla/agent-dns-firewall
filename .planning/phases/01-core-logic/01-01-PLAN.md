<task type="auto" tdd="true">
  <name>Task 2: Normalization function and parser implementations</name>
  <files>src/normalize.ts, src/parse.ts, tests/normalize.test.ts, tests/parse.test.ts</files>
  <action>
1. Create `src/normalize.ts`:
   - Export `normalizeDomain(raw: string): string` that trims whitespace, lowercases, and strips trailing dot.

2. Create `src/parse.ts`:
   - Import `normalizeDomain` from `./normalize.js` (use .js extension for ESM).
   - Define `IP_PATTERN = /^(\d{1,3}\.){3}\d{1,3}$|^[0:]+[01]?$/` to match IPv4 and common IPv6 loopback patterns.
   - Define `SPECIAL_HOSTS` Set containing: `localhost`, `localhost.localdomain`, `local`, `broadcasthost`, `ip6-localhost`, `ip6-loopback`, `ip6-localnet`, `ip6-mcastprefix`, `ip6-allnodes`, `ip6-allrouters`, `ip6-allhosts`.
   - Export `parseHostsFormat(content: string): string[]`:
     - Split on `\r?\n`
     - For each line: strip inline comments (`line.split('#')[0]`), trim, skip empty
     - Split remaining on whitespace, for each token: skip if matches IP_PATTERN, normalize, skip if empty or in SPECIAL_HOSTS, push to result
   - Export `parseDomainList(content: string): string[]`:
     - Split on `\r?\n`
     - For each line: strip inline comments (`line.split('#')[0]`), trim, skip empty
     - Normalize line, skip if empty, push to result

3. Create tests FIRST (TDD red-green):

   `tests/normalize.test.ts`:
   - Lowercases: `"EXAMPLE.COM"` -> `"example.com"`
   - Trims: `"  example.com  "` -> `"example.com"`
   - Strips trailing dot: `"example.com."` -> `"example.com"`
   - Handles all three together: `"  EXAMPLE.COM.  "` -> `"example.com"`
   - Empty string returns empty string
   - Already normalized passes through unchanged

   `tests/parse.test.ts`:
   - **hosts format:**
     - Parses standard line: `"0.0.0.0 ad.example.com"` -> `["ad.example.com"]`
     - Parses 127.0.0.1 prefix: `"127.0.0.1 ad.example.com"` -> `["ad.example.com"]`
     - Skips comment lines: `"# comment\n0.0.0.0 ad.com"` -> `["ad.com"]`
     - Strips inline comments: `"0.0.0.0 ad.com # tracking"` -> `["ad.com"]`
     - Skips blank lines: `"\n\n0.0.0.0 ad.com\n\n"` -> `["ad.com"]`
     - Handles multi-domain lines: `"0.0.0.0 ad1.com ad2.com"` -> `["ad1.com", "ad2.com"]`
     - Filters IP-only lines: `"0.0.0.0"` -> `[]`
     - Filters special hostnames: `"0.0.0.0 localhost"` -> `[]`
     - Filters localhost.localdomain: `"0.0.0.0 localhost.localdomain"` -> `[]`
     - Normalizes to lowercase: `"0.0.0.0 AD.COM"` -> `["ad.com"]`
     - Handles `\r\n` line endings
     - Handles mixed IPv6 `::1` prefix: `"::1 localhost"` -> `[]` (both IP and special host filtered)
   - **domain-list format:**
     - Parses one domain per line: `"ad.com\ntracker.com"` -> `["ad.com", "tracker.com"]`
     - Skips comment lines: `"# list\nad.com"` -> `["ad.com"]`
     - Strips inline comments: `"ad.com # bad"` -> `["ad.com"]`
     - Skips blank lines
     - Normalizes to lowercase
     - Handles `\r\n` line endings
  </action>
