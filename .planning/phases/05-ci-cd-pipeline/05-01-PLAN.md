Create `.github/workflows/` directory and `ci.yml` with:

**Workflow-level config:**
- `name: CI`
- `permissions: contents: read` (least privilege)
- Triggers: `push` (all branches) + `pull_request`, both with `paths-ignore`:
  - `'**/*.md'`
  - `'.planning/**'`
  - `'docs/**'`
  - `'LICENSE'`
- Concurrency: `group: ci-${{ github.ref }}`, `cancel-in-progress: true`

**Job 1 — `test`:**
- `runs-on: ubuntu-latest`
- `strategy.fail-fast: false`
- `strategy.matrix.node-version: [18, 20, 22]`
- Steps:
  1. `actions/checkout@v4`
  2. `actions/setup-node@v4` with `node-version: ${{ matrix.node-version }}` and `cache: 'npm'`
  3. `run: npm ci`
  4. `run: npm test`

**Job 2 — `validate`:**
- `needs: [test]`
- `runs-on: ubuntu-latest`
- Steps:
  1. `actions/checkout@v4`
  2. `actions/setup-node@v4` with `node-version: 22` and `cache: 'npm'`
  3. `run: npm ci`
  4. `run: npm run check`

Use the exact YAML structure from the RESEARCH.md architecture pattern. All decisions are locked — no deviations.
