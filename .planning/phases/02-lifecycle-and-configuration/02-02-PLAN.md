2. Create src/firewall.ts (GREEN):
   - Export `createDomainFirewall(config: FirewallConfig): DomainFirewall`
   - Use closure-based factory pattern (not class). Internal mutable state:
     - `blocklistEntries: Array<{ index: Set<string>; sourceId: string }>` -- starts empty
     - `refreshTimer: ReturnType<typeof setTimeout> | null` -- starts null
     - `abortController: AbortController | null` -- starts null
   - Build `allowSet` and `denySet` from config.allow/config.deny using `buildDomainIndex` (computed once in factory)
   - `defaultLog` function: routes 'error' to console.error, 'warn' to console.warn
   - `log` = config.log ?? defaultLog
   - `start()`:
     - Call stop() first (idempotent -- clears old state)
     - Create new AbortController
     - Call fetchAllSources with config.sources, signal, log
     - Assign result to blocklistEntries (atomic swap)
     - Call scheduleRefresh()
   - `stop()`:
     - Abort controller if exists, set to null
     - Clear refresh timer if exists, set to null
   - `isDomainBlockedFn(domain)`:
     - Delegate to imported isDomainBlocked(domain, allowSet, denySet, blocklistEntries)
   - `scheduleRefresh()`:
     - If no config.refreshMinutes, return
     - Set refreshTimer = setTimeout(async callback, refreshMinutes * 60_000)
     - In callback: try fetchAllSources with current abortController signal. On success, swap blocklistEntries. On failure (e.g., abort), preserve existing entries. Then scheduleRefresh() to chain next.
   - Return `{ start, stop, isDomainBlocked: isDomainBlockedFn }`

Run tests: `npx vitest run tests/firewall.test.ts -x`
