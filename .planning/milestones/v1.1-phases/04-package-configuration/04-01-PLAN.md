<task type="auto">
  <name>Task 2: Validate package correctness with full check pipeline</name>
  <files>package.json</files>
  <action>
Run the full validation pipeline to confirm the package is correctly shaped for npm publishing:

1. **Build:** `npm run build` -- should produce dist/ with .js, .d.ts, and .d.ts.map files (no .js.map files)
2. **publint:** `npx publint` -- should pass with no errors (VAL-01)
3. **attw:** `npx attw --pack` -- should pass with no errors, confirming types resolve under all moduleResolution modes (VAL-02)
4. **Tarball contents:** `npm pack --dry-run` -- verify output shows ONLY:
   - dist/**/*.js
   - dist/**/*.d.ts
   - dist/**/*.d.ts.map
   - LICENSE
   - README.md
   - package.json
   - NO src/ files, NO test files, NO config files (tsconfig, vitest.config, etc.)
5. **Publish dry run:** `npm publish --dry-run` -- confirms prepublishOnly triggers the full check chain (BUILD-02)

If publint or attw report warnings (not errors), assess whether they are actionable. Fix any errors by adjusting package.json or tsconfig.json as needed.

If `npm pack --dry-run` shows unexpected files, check that `"files": ["dist"]` is correctly set.

If attw reports issues with specific moduleResolution modes, the most likely cause is the exports field shape. Ensure `types` is the first condition in each export entry.
